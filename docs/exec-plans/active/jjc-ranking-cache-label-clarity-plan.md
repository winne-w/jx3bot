# JJC 排名角色弹窗缓存文案澄清计划

## 背景

排名统计页面点击角色后，角色近期对局弹窗同时展示近期对局列表缓存、赛季战绩缓存；继续点开单场对局后还会展示对局详情缓存。当前文案都使用“缓存时间”“缓存/更新”，用户难以判断分别对应哪份数据。

## 目标

- 明确区分角色弹窗中的“近期对局数据缓存”和“赛季战绩数据缓存”。
- 明确区分单场对局弹窗中的“对局详情数据缓存”。
- 去掉角色弹窗副标题中的“最近 3v3 战绩窗口”说明，把近期对局缓存时间移到对局列表附近。
- 近期对局支持手动刷新，并在刷新时绕过服务端第一页缓存。

## 涉及文件

- `public/jjc-ranking-stats.html`
- `src/api/routers/jjc_ranking_stats.py`
- `src/services/jx3/jjc_ranking_inspect.py`
- `tests/test_jjc_ranking_inspect.py`

## 实施步骤

1. 将角色近期对局弹窗副标题中的缓存文案改为“近期对局数据：缓存于/实时获取”。
2. 将战绩概览区域标签改为“赛季战绩：缓存于/刚更新于”，避免与近期对局缓存混淆。
3. 将单场对局详情弹窗副标题改为“对局详情数据：缓存于/实时获取”。
4. 去掉角色弹窗副标题中的窗口说明，在对局列表标题栏展示对局缓存时间与“刷新对局”按钮。
5. 给 `role-recent` API 与 service 增加 `force_refresh`，刷新第一页时绕过 Mongo 缓存并写回新缓存。
6. 运行 HTML/JS 静态检查与相关 Python 单测；该页面为静态前端，最终仍需浏览器手工确认弹窗文案。

## 验证方式

- 执行可用的语法检查命令，确认改动未引入明显语法错误。
- 手工回归路径：打开 `public/jjc-ranking-stats.html`，点击角色查看近期对局，确认顶部副标题与指标标签分别标注数据来源；再点击单场对局，确认对局详情副标题标注数据来源。
- 点击“刷新对局”，确认请求 `/api/jjc/ranking-stats/role-recent` 带 `force_refresh=true`，列表和“对局列表”缓存时间更新。
